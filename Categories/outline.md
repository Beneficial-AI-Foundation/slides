1. from set theory as collections of elements, to elements as functions from * to a set.
2. in programming languages and type theory, types are not collections of elements, but a symbol with constructors (functions into the type).
3. in algebra, we think of matrices as linear maps (functions) between fixed dimensional spaces. If input and output dimensions are the same, then we have the ring of square matrices. But if input/output dims are different, they don't form a ring. 
4. Instead, they form a category, with natural numbers as objects, and the matrices as morphisms. you can only "multiply" or compose them if the dimensions match up
